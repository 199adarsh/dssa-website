Hackathon Guidelines

Understand the rules, form your team, and prepare to innovate. Your journey starts here.

Note: Participants are expected to build unique solutions. Submissions copied from GitHub or pre-existing projects will be disqualified.

Team Formation

  • Teams: 2–4 people (solo not allowed)
  • One team per person, one project per team
  • No cross-participation with the same project in multiple hackathons

Code of Conduct

  • Respectful, harassment-free environment
  • No hate speech, abuse, discrimination
  • Everyone (participants, mentors, judges) must follow it

Project Development

  • Must be built during hackathon (no pre-built projects)
  • Allowed: open-source libraries, frameworks, public APIs
  • Clearly separate what you built vs. reused
  • Must align with hackathon theme/tracks
  • No plagiarism or copying others' work

Tools, Datasets & APIs

  • Any programming languages, IDEs, frameworks allowed
  • Public APIs and cloud services usually allowed (with credit)
  • All AI tools allowed for building projects
  • During judging, you must explain your code implementation
  • Illegal/pirated tools = strictly forbidden
